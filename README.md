# Flight Visualizer ✈️

A local, **fully offline** dashboard for your flight history. Point it at a flight-logs CSV (e.g. a [Flighty](https://flighty.com/) export) and it renders 20 cards of stats — maps, routes, airports, countries, layovers, records, and more. Everything runs in your browser; **no data ever leaves your machine** and there are no network calls at runtime.

## Highlights

- **Offline single file** — `npm run build` produces one self-contained `dist/index.html` you can double-click (`file://`), no server, no internet.
- **Metro grouping** — nearby airports collapse into metros ("Dallas (DFW/DAL)"), editable in `src/reference/airport-groups.json`.
- **Timezone-aware** — durations and layover gaps are computed from per-airport IANA timezones (raw timestamps are local wall-clock).
- **All-time + per-year** views, with a settings panel (group airports, route directionality, include-canceled, exclude-before-date, **home base**, layover threshold).
- **Click-through** — click any country / airport / airline / route / layover to see the underlying flights, then drill into a Flight Detail.
- **20 cards**: world map, overview, odometer, airports, routes, common layovers, airlines, distance, countries (+ US/India/Mexico state split), super-domestic, intercontinental, geographic extremes, when-you-fly, travel intensity, aircraft, same-metal (tail numbers), records, delays, shortest, longest.

## Run it

```bash
npm install
npm run dev      # local dev server
npm test         # vitest (246 tests)
npm run build    # → dist/index.html (single offline file)
```

Then drop your flight-logs CSV onto the page.

## Privacy

The app reads your CSV in-browser only. Your real export is **git-ignored** (`src/test/fixtures/real-sample.csv`); the only committed fixtures are tiny synthetic ones for tests.

## Tech

Vite · React + TypeScript · Vitest · Luxon (timezones) · D3-geo + world-atlas (offline map). Reference airport/airline data is bundled (generated by `scripts/preprocess/`).
